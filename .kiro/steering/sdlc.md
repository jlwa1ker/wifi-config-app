# Software Development Lifecycle

## Development Modes

There are two modes of development. The current mode determines how bugs are handled when discovered.

### Rapid Application Development (RAD)

- Fix bugs immediately as they are found
- No issue tracking required
- Focus on velocity and iteration
- Appropriate during initial feature buildout or prototyping

### Stability

- File a GitHub issue whenever a bug is found
- Do not fix bugs inline — queue them for later
- Issues are prioritized and addressed in subsequent fix branches
- Appropriate when the system is deployed and reliability matters
